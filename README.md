### Project topic


Sugarcane (Saccharum spp.) is a monocotyledonous plant from the Poaceae family, native to Papua New Guinea. The Saccharum genus includes several species, with modern cultivars being complex interspecific hybrids primarily derived from crosses between S. officinarum and S. spontaneum. Genetically, sugarcane possesses one of the most complex genomes among cultivated plants: polyploid and aneuploid, with chromosome numbers ranging from 2n = 80 to 2n = 130 depending on varieties, and an estimated genome size of approximately 10 Gb.
Sugarcane represents one of the world's most strategic crops for food, energy, and industrial purposes. Globally, it ranks as the primary sugar crop and a major source of plant-based renewable energy, with production estimated at over 1.93 billion tons in 2024. In Côte d'Ivoire, sugarcane ranks 6th among food and industrial crops with production of approximately 2.15 million tons.
However, sugarcane productivity is limited by numerous abiotic and biotic constraints. Among biotic constraints, viral diseases such as mosaic, primarily caused by Sugarcane mosaic virus (SCMV), represent major threats. In Côte d'Ivoire, symptoms identical to mosaic have been observed on several sugarcane varieties. Until now, virtually no studies have been conducted in Côte d'Ivoire on the complete microbiome associated with these symptoms.
Traditional pathogen identification methods (serology, targeted PCR), while effective for detecting known organisms, have significant limitations: they cannot discover new pathogens or characterize complex infections involving multiple microorganisms simultaneously.
In this context, metagenomics emerges as an essential approach. This discipline uses high-throughput sequencing to analyze all genetic material present in an environmental sample without requiring prior cultivation. Among next-generation sequencing technologies, Nanopore sequencing distinguishes itself by generating long nucleotide sequences, facilitating genome assembly and classification.
In this internship, the analyzed data were generated by Nanopore sequencing from sugarcane leaves showing mosaic symptoms, collected from different sites in Côte d'Ivoire. Our role is to exploit these data to achieve complete characterization of all genetic entities present in these samples: viruses, bacteria, fungi, and any other detectable genetic entity, whether known, emerging, or undescribed.

#!/bin/bash

# Source and destination folder

SOURCE_DIR="/scratch/noe/sugarcane_projet/raw_data/Metagenomics/sugarecane_demux"
Dest_dir="/scratch/noe/sugarcane_projet/raw_data"

# Create the destination folder for each barcode.
mkdir -p "$Dest_dir"

# Loop on the barcodes
for b in {75..85}; do
    # Create the specific folder for the barcode
    TARGET_DIR="$Dest_dir/barcode${b}"
    mkdir -p "$TARGET_DIR"

    echo "Barcode processing ${b}..."
# Let's find all the FastQ files for the barcodes in the source folder and then copy those files into 
    # in the destination folder in the corresponding barcodes 
    find "$SOURCE_DIR" -type f -name "*barcode${b}*.fastq" | while read -r file; do
        # We extract the name of the run (ex: AQN321)
      
        BASE_NAME=$(basename "$file")

        # Copying and renaming the file 
        cp "$file" "$TARGET_DIR/${BASE_NAME}"
    done
done
```

```
#!/bin/bash

### set the job name
#SBATCH --job-name=Minimap2_and_kraken

### Set the number of CPUs to use 
#SBATCH --cpus-per-task=8

###Total RAM allocated (for step of mapping and kraken)
#SBATCH --mem=30G      

### Set the partition to use
#SBATCH --partition=normal  
 
###Specify the node on which the job should run
#SBATCH --nodelist=node02 

### error file and file output (If an error occurs, I can check the .err file to understand the problem)
#SBATCH --output=/scratch/noe/sugarcane_projet/logs/Minimap2_and_kraken_%j.out 
#SBATCH --error=/scratch/noe/sugarcane_projet/logs/Minimap2_and_kraken_%j.err  


# Let's define the variables
REF="/scratch/noe/sugarcane_projet/banks/reference_cat_sugarcane/genome_ref.fna"
RAW_DATA="/scratch/noe/sugarcane_projet/raw_data/merged_fastq"
CLEANING_DIR="/scratch/noe/sugarcane_projet/results/2_mapping"
DB_KRAKEN="/scratch/noe/sugarcane_projet/banks/db_kraken"
OUT_KRAKEN="/scratch/noe/sugarcane_projet/results/3_kraken_read_unmapped"

# Let's load the modules
module load  bioinfo-wave
module load minimap2/2.30.0
module load samtools/1.19.2
module load kraken2/2.17.1

# Let's create recursive directories
mkdir -p "$CLEANING_DIR"
mkdir -p "$OUT_KRAKEN"


for file in "${RAW_DATA}"/*.fastq; do     # I create a loop that iterates through the files in the RAW_DATA variable, and on each iteration I store the file name in a temporary variable called file 
  names=$(basename "$file" .fastq)        # To retrieve the filename without the .fastq extension, we then assign it to a variable called names       
  echo "start of analysis of $names"    
  # Filtering reads using the reference genome
  echo "Mapping reads to the reference genome for the samples $names"
  minimap2 -I 10G -t "$SLURM_CPUS_PER_TASK" -ax map-ont "$REF" "$file" -o "${CLEANING_DIR}/${names}_mapped.sam"  # because the sugarcane genome is approximately 10GB in size and minimap uses -I 8G by default 
  echo "generation of mapping statistics (samtools flagstat) for $names"
  samtools flagstat "${CLEANING_DIR}/${names}_mapped.sam" > "${CLEANING_DIR}/$names.stats"
  echo "retrieval of unmapped reads for $names, conversion from SAM to BAM (samtools view) and bam in FASTQ (samtools fastq) "
  samtools view -@ "$SLURM_CPUS_PER_TASK" -f 4 -b "${CLEANING_DIR}/${names}_mapped.sam" > "${CLEANING_DIR}/${names}_unmapped.bam"
  samtools fastq "${CLEANING_DIR}/${names}_unmapped.bam" > "${CLEANING_DIR}/${names}_unmapped.fastq"
  # Taxonomic classification of unmapped reads using Kraken2 with the k2_pluspfp_16_GB database
  echo "kraken2 de $names"
  kraken2 --db "$DB_KRAKEN" "${CLEANING_DIR}/${names}_unmapped.fastq" --threads "$SLURM_CPUS_PER_TASK" --report "${OUT_KRAKEN}/${names}_kraken_report.txt" --output "${OUT_KRAKEN}/${names}_kraken_output.txt"
  echo "end of the analysis"
done

#!/bin/bash

### Set the job name
#SBATCH --job-name=Polishing

### Set the number of CPUs to use 
#SBATCH --cpus-per-task=4

### Set the partition to use
#SBATCH --partition=normal

#SBATCH --nodelist=node02

### error file and file output
#SBATCH --output=/scratch/noe/sugarcane_projet/logs/polishing_%j.out
#SBATCH --error=/scratch/noe/sugarcane_projet/logs/polishing_%j.err


# Let's define the variables 
READS="/scratch/noe/sugarcane_projet/results/2_mapping/"
CONTIGS="/scratch/noe/sugarcane_projet/results/5_polishing/input_contigs/"
OUTDIR="/scratch/noe/sugarcane_projet/results/5_polishing"
OUTDIR_RACON="/scratch/noe/sugarcane_projet/results/5_polishing/Racon_polishing"

mkdir -p "$OUTDIR"
mkdir -p "$OUTDIR_RACON"

# download the modules 
module load bioinfo-wave
module load minimap2/2.30.0 
module load racon/1.4.20 

# Creating a loop 
for file in "$READS"/*_unmapped.fastq; do
  # Retrieve the file names and store them in a new variable without the extension _unmapped.fastq
  names=$(basename $file _unmapped.fastq)
  echo "Polishing of $names"
  # Let’s retrieve the files from the CONTIGS variable and then store them in a new variable called contigs 
  contigs="$CONTIGS/${names}_contigs.fasta"
  # Mapping unmapped reads to contigs 
  minimap2 -t $SLURM_CPUS_PER_TASK -x map-ont "$contigs" "$file" > "$OUTDIR/${names}_backmapping.paf" 
  # Let's launch racon
  racon -t $SLURM_CPUS_PER_TASK "$file" "$OUTDIR/${names}_backmapping.paf" "$contigs" > "$OUTDIR_RACON/${names}_racon_polishing.fasta"
  echo "End of polishing de $names"
done

#!/bin/bash

### Set the job name
#SBATCH --job-name=Kraken_Classification

### Set the number of CPUs to use

### Set the partition to use
#SBATCH --partition=normal

### error file and file output
#SBATCH --output=/scratch/noe/sugarcane_projet/logs/Kraken_Classification_%j.out
#SBATCH --error=/scratch/noe/sugarcane_projet/logs/Kraken_Classification_%j.err

# Download the module
module load bioinfo-shared
module load kraken2/2.1.2
module load krona

# Let's define the variables
CONTIGS_POLISHING="/scratch/noe/sugarcane_projet/results/5_polishing/Racon_polishing"
OUT_DIR="/scratch/noe/sugarcane_projet/results/6_kraken_contigs_polishing"
DB_KRAKEN="/scratch/noe/sugarcane_projet/banks/db_kraken"

mkdir -p "$OUT_DIR"

for file in "$CONTIGS_POLISHING"/*_polishing.fasta; do
   names=$(basename $file _polishing.fasta)
   echo "Kraken of $names"

   # Fix fasta headers to remove everything after the ID
   fixed_file="${OUT_DIR}/${names}_headers_fixed.fasta"
   awk '{print $1}' "$file" > "$fixed_file"

   # Classification taxonmique kraken
   kraken2 --db "$DB_KRAKEN" "$fixed_file" --threads $SLURM_CPUS_PER_TASK --report "$OUT_DIR/${names}_kraken_report" --output "$OUT_DIR/${names}_kraken_output"
done
